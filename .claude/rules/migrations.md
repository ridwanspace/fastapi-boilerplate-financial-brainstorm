# Migration Rules

- Always use `--autogenerate` from SQLAlchemy models — never hand-write DDL for model changes
- Migration command: `make migrate-create msg="description"` (wraps `alembic revision --autogenerate`)
- Apply migrations: `make migrate` (wraps `alembic upgrade head`)
- Check `PRD.md` for required columns on financial tables (version, audit fields, soft-delete, etc.)
- **Never drop a column** without confirming it is unused in all environments
- For renaming: always add new column → backfill → switch reads → remove old column (three migrations)
- Review autogenerated migration before applying — check constraints and indexes
- Migration filenames use the auto-generated revision prefix — do not rename them
